# CUDA 2D 归一化算子性能与精度测试报告

本报告记录了针对视觉张量 (B, C, H, W) 优化的 CUDA 融合归一化算子 (`RMSNorm2d` 和 `LayerNorm2d`) 的测试结果。

## 1. 精度验证 (Correctness)

测试配置: `B=16, C=256, H=16, W=16`, 数据类型: `FP32`。

| 算子类型 | 测试项 | 误差 (Max Diff) | 状态 |
| :--- | :--- | :--- | :--- |

| | 反向传播 (Bwd) | 0.00000083 | ✅ 通过 |
| **LayerNorm2d**| 前向传播 (Fwd) | 0.00000143 | ✅ 通过 |
| | 反向传播 (Bwd) | 0.00000061 | ✅ 通过 |
| | 权重梯度 (dW) | 0.00004578 | ✅ 通过 |
| | 偏置梯度 (dB) | 0.00000000 | ✅ 通过 |

### 精度分析
- **FP32 机器精度:** $10^{-6}$ 级别的 Fwd/Bwd 误差属于正常的浮点数舍入误差，源于 CUDA 融合算子与 PyTorch 原生实现（多步操作）在计算顺序上的微小差异。
- **梯度累加误差:** $dW$ 误差稍大 ($10^{-5}$) 是由于在反向传播中对大量空间位置 ($16 \times 16 \times 16 = 4096$ 个元素) 进行累加导致的数值偏移，符合工业级深度学习算子开发标准。

## 3. 核心优势分析

1. **Kernel Fusion (内核融合):** 
   - 将“计算均值/方差”、“减去均值”、“除以标准差”、“乘以权重”、“加上偏置”等多个操作合并为一个 CUDA Kernel。
   - 避免了中间临时张量 (Intermediate Tensors) 的生成，大幅降低了访存开销和 Kernel Launch 延迟。
2. **2D Native Optimization:**
   - 相比 PyTorch 官方常用的 `Permute(B,C,H,W -> B,H,W,C) + 1D LayerNorm + PermuteBack` 流程，我们的算子直接在 2D 布局上进行归约累加，避免了昂贵的 `Permute` (转置) 操作带来的数据重排开销。
3. **显存效率:**
   - 峰值显存显著降低，尤其是在反向传播中，融合算子只需保存更少的激活值用于计算梯度。
